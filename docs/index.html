Lictor

Claude Code / OpenAI Codex などの エージェント TUI CLI をラップする per-session sidecar。 セッション内のフックがホスト端末を操作し（ウィンドウタイトル）、TUI にキー入力を注入し、 Concordia（マルチエージェント調整サービス）と連携できるようにする薄いラッパ。

サービスの役割

Claude Code の TUI は、自分が起動したサブプロセス（Bash ツール / フック / MCP）の標準出力を 取り込んでしまう。そのため、サブプロセスが発行した OSC エスケープ（\e]0;TITLE\a）は レンダラに吸収され端末に届かない —— セッション内から端末タイトルを変えられない という既知の問題（anthropics/claude-code #15802, #18326, #20441）がある。

Lictor は claude の親プロセスになる ことでこれを回避する。Lictor 自身の process.stdout が本物の端末 pty なので、Lictor が発行する OSC は Windows Terminal / iTerm2 / Alacritty にそのまま届く。セッション内のサブプロセスは loopback HTTP sidecar 経由で Lictor に依頼し、実際のエスケープ書き込みは Lictor が行う。

核となる不変条件（drop-in wrapper） Lictor の process.stdout だけが本物の端末 pty に繋がっている。フックは claude の stdout を通さず、loopback でこの sidecar に到達する。

設計

なぜ HTTP loopback + pty なのか

  • 移植性 — HTTP-over-TCP loopback は Windows / macOS / Linux で同一に動く（OS 固有のソケットパス不要）。
  • Concordia との対称性 — Concordia が既に HTTP なので proxy 経路が fetch in / fetch out で済む。
  • フックの使いやすさ — curl はどこにでもある。named pipe は OS ごとにツールが違う。
  • pty.spawn の採用（v0.3） — sidecar が TUI に入力を注入するには pty マスターである必要がある。prebuild 同梱の node-pty@^1.1 のみ採用（gyp ビルド不要）。

2 つの trust boundary

ユーザ供給テキストをホスト端末 / TUI に渡す関数は 2 つだけで、両方とも先にサニタイズする。

境界関数処理
OSC タイトルsanitizeTitle (src/osc.ts)C0/DEL 除去・200 文字 cap・多バイト保持。OSC の早期終端や別エスケープ注入を防ぐ。
pty 注入sanitizeRenameArg (src/sidecar.ts)C0/DEL 除去・先頭 / 除去（slash チェイン防止）・trim・長さ cap。

ステートレス設計

Lictor は永続データを持たない per-session ラッパ（状態は Concordia / env / pty 側）。セッション専用 ディレクトリは終了時に丸ごと削除され、グローバルな ~/.claude/skills/ を汚さない。 Concordia 不在時は v0.0 相当に劣化し、中継系は 500 ではなく 503 を返す。

アーキテクチャ図と設計判断の全文は DESIGN.md を参照。

仕様（spec）とコードの整合性レビュー

① Local LLM が外部 Famulus へ移行（spec/README が陳腐化） spec/local-llm-agent.md と README は内蔵 REPL（binary = "lictor", spawnArgs = ["cli","local-agent"]）を記述するが、src/provider.ts の gemma4-12 は外部バイナリ famulus run（binary = "famulus"）へ 移行済み（2026-06-10）。src/local-agent/ は旧実装として残存。
② fs-rpc が spec 未記載 GET /v1/fs/read | /v1/fs/list | /v1/fs/grep（cwd 限定の read/list/grep）は実装 （src/fs-rpc.ts）+ テスト（tests/fs-rpc.test.ts）済みだが、 spec/interface/sidecar-http-api.md の表にも feature 索引にも存在しない。
③ /v1/internal/* が spec API 表に不在 /v1/internal/permission-check / permission-response / ask-question / force-exit は実装済み。force-exit は README にあるが spec の API 表には未掲載。
④ askquestion-pending-gate.md が feature 索引から漏れ spec/feature/askquestion-pending-gate.md は存在するが、spec/README.md の feature 一覧表に行が無い（9 行掲載／実体 10 ファイル）。
⑤ Gemini プロバイダが README プロバイダ表に不在 src/provider.ts に gemini（displayName: "Gemini CLI"）が あるが、README のプロバイダ表は claude / codex / local のみ。
